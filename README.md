# reports

日報 → 週次進捗 → 月次成果報告 を一貫して管理するディレクトリ。
プレーンMarkdown + ウィキリンク `[[]]` 構成。後でこのフォルダをそのまま
Obsidian Vault として開けば移行完了（`.obsidian/` が増えるだけ）。

## ディレクトリ

```
reports/
├── templates/      daily / weekly / monthly のテンプレ
├── daily/YYYY/MM/  日報（1日1ファイル）
├── weekly/YYYY/    週次（ISO週・1週1ファイル）
├── monthly/YYYY/   月次（1月1ファイル）
└── _attachments/   画像・添付
```

## 命名規則

| 層 | ファイル名 | 例 |
|----|-----------|-----|
| 日報 | `YYYY-MM-DD.md` | `2026-06-25.md` |
| 週次 | `YYYY-Www.md`（ISO週番号） | `2026-W26.md` |
| 月次 | `YYYY-MM.md` | `2026-06.md` |

ISO週番号は `date +%G-W%V` で確認できる。

## 運用フロー

1. **毎日**: `templates/daily.md` をコピーして `daily/YYYY/MM/` に作成。
   - Teams貼り付け前提フォーマット（`#`/`[]()`/`**` は使わない。【本日の進捗】/【明日の予定】/【気づき・相談】の3セクション、チケットは行末 `(URL)`）。詳細は [CLAUDE.md](./CLAUDE.md)。
2. **週末**: その週の日報5本を読み、`templates/weekly.md` から週次を作成。
   - 「対象の日報」に `[[YYYY-MM-DD]]` を列挙。
   - 日々のログ → タスク単位の進捗へ変換。
3. **月末**: その月の週次を集約し、`templates/monthly.md` から月次を作成。
   - 各週の `★`・達成項目だけを拾い、**成果・価値**に翻訳する（作業の羅列にしない）。

## リンクで繋ぐ

- 週次に当該日報を `[[2026-06-22]] [[2026-06-23]]...`
- 月次に当該週次を `[[2026-W26]] [[2026-W27]]...`

これで日報→週次→月次がリンクで辿れる。Obsidian化後はグラフ／バックリンクが効く。

## Obsidian移行時

- このフォルダを Vault として開く。
- テンプレ: コア「テンプレート」or Templater プラグインの参照先を `templates/` に。
- 添付先を `_attachments/` に設定。
- デイリーノート機能を使うなら保存先を `daily/`、形式を `YYYY/MM/YYYY-MM-DD` に。
