# CLAUDE.md — reports

日報 → 週次進捗 → 月次成果報告 を管理するディレクトリ。
ユーザーは生ログを投げ、整形・集約は Claude が代行する運用。
詳細な運用ルールは [README.md](./README.md) を参照。

## このディレクトリでの役割

ユーザーが「今日やったこと」を投げてきたら、日報の作成・整形を代行する。
週末・月末は下位の層を集約して上位の報告を作る。

## 命名規則（厳守）

| 層 | パス | 例 |
|----|------|-----|
| 日報 | `daily/YYYY/MM/YYYY-MM-DD.md` | `daily/2026/06/2026-06-25.md` |
| 週次 | `weekly/YYYY/YYYY-Www.md` | `weekly/2026/2026-W26.md` |
| 月次 | `monthly/YYYY/YYYY-MM.md` | `monthly/2026/2026-06.md` |

- 週番号は **ISO週**。`date -d <YYYY-MM-DD> +%G-W%V` で確定させる（推測しない）。
- 曜日も `date` で確定させる。

## 日報を作るとき

**Teams貼り付け前提フォーマット**（ユーザーは日報を Teams にそのまま貼るため、
Markdown 記号が生表示になるのを避ける）。

1. `templates/daily.md` をベースにする。
2. frontmatter の `date` / `week` を `date` コマンドで埋める。
3. 本文は `■ タイトル` 行以降に3セクションで書く: **【本日の進捗】/【明日の予定】/【気づき・相談】**。
   ユーザーの生ログをこの3つに振り分ける（課題・ブロッカーは【気づき・相談】に入れる）。
4. 記法ルール（厳守）:
   - `#` 見出し / `[]()` Markdownリンク / `**` は使わない。
   - セクションは `【】`、箇条書きは `・`、サブ項目は全角スペース＋`-`。
   - チケットURLはタイトル同一行の末尾 `(URL)` に置く（改行しない。Teamsが自動リンク化）。
   - 項目間は空行を1つ入れて行間を確保する。
5. 投げられた事実だけを書く。やってないことを補完・創作しない。

## 週次を作るとき

1. 対象週の日報（`daily/YYYY/MM/` 内で当該ISO週の日付）を全て読む。
2. `templates/weekly.md` をベースに、日々のログをタスク単位の進捗へ変換。
3. 「対象の日報」に `[[YYYY-MM-DD]]` でリンクを列挙。
4. 前週の週次があれば「来週の計画」を読み、予実（達成状況）を埋める。

## 月次を作るとき

前提: ユーザーは**業務委託（請負＝成果物ベース）**。会社向けの月次成果報告として作る。
`templates/monthly.md` をベースにする。

1. 素材を集める: 対象月の週次（`weekly/YYYY/`）を全て読む。週次が無ければ日報、
   それも無ければ生ログ（Teams / Backlog 等、`_attachments/YYYY-MM/`）から直接集約する。
2. **成果物ベースで書く**。冒頭の「当月の成果物・納品状況」表を主役にし、各成果物に
   状態（完了 / リリース済 / 進行中 / 未着手）を付ける。**作業の羅列にしない**＝成果・価値・
   数値・影響に翻訳する（日報/週次と決定的に違う層）。
3. **工数・稼働時間は載せない**（請負のため）。指標は件数・対応量・実測値で。
4. 「来月の予定・目標」は **「対象: 達成状態」の目標形式**で書く（例: `須藤さんscript 追加実装: 実装完了`）。
5. 日報にある私的所感（モチベ・体調等）はクライアント向け月次に混ぜない。
6. 同一名でも系統が違う成果物（例: 面積採用ロジックの sudo-judge側 と 算定アプリ側）は
   取り違えず、状態を別々に書く。
7. 生ログを使った場合は `_attachments/YYYY-MM/` に置き、必要なら参照を残す。
8. **PDF化を指示されたとき**のみPDFを書き出す（指示があるまで生成しない）。出力先は
   `monthly/YYYY/exports/`、ファイル名は `YYYYMM_業務報告書_飯田正輝.pdf` 形式
   （例: `monthly/2026/exports/202606_業務報告書_飯田正輝.pdf`）。html等の中間生成物も
   `exports/` に置く。`exports/` はフォルダごと `.gitignore` 済みでコミットしない。

## やらないこと

- 事実の創作・水増し。投げられていない成果を勝手に足さない。
- 命名規則・ディレクトリ構成の逸脱。
- 自動実行の前提（このセッションが動いている間だけ動く。勝手に走らない）。
